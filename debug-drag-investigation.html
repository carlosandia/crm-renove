🔍 Investigação: Por que cards não reconhecem todas as etapas?

📋 ROTEIRO DE INVESTIGAÇÃO

Este roteiro irá ajudar a identificar por que o drag & drop só reconhece a etapa "Lead" ao invés de todas as etapas.

🚀 PASSO 1: Acesse o Pipeline

Navegue até um pipeline com múltiplas etapas no CRM (porta 8080)

http://localhost:8080 → Pipeline → Selecionar pipeline

🔧 PASSO 2: Abra o Console do Browser

Pressione F12 e vá para a aba Console

🎯 PASSO 3: Inicie um Drag

Arraste qualquer card. O debug será ativado automaticamente e você verá logs como:

🔍 [DRAG START] Debug ativado! Investigando drag do lead: xxx
🔍 [DRAG START] Encontradas X etapas na página:
🔍 [RENDER] Renderizando X etapas:
🔍 [INVESTIGAÇÃO] Total containers detectados: X

📊 PASSO 4: Analise os Logs

Procure por estes indicadores de problema:

  • ❌ REJEITADO: type undefined !== 'stage' - Container sem tipo
  • ❌ REJEITADO: container.disabled = true - Etapa desabilitada
  • disabled: true nos logs do KanbanColumn
  • Diferença entre "Encontradas X etapas" e "X etapas válidas"

🔍 PASSO 5: Verificar Configuração das Etapas

Nos logs, compare:

  • Etapas Renderizadas: Quantas aparecem no [RENDER]
  • Containers Detectados: Quantos no [INVESTIGAÇÃO]
  • Containers Válidos: Quantos no [RESULTADO]

⚠️ POSSÍVEIS DIAGNÓSTICOS

  • Se containers detectados < etapas renderizadas: Problema de timing/renderização
  • Se containers válidos < containers detectados: Problema de validação (disabled ou type)
  • Se só "Lead" aparece como válido: Outras etapas têm isDropDisabled = true

📝 PASSO 6: Copie os Logs

Copie todos os logs que começam com 🔍 para análise detalhada

🎯 OBJETIVO

Identificar exatamente qual validação está falhando e por que apenas uma etapa aceita drops.
