✅ Correção Aplicada - Modal Campos Customizados

CORREÇÃO CONCLUÍDA! O problema do modal que fechava automaticamente ao adicionar/editar campos customizados foi identificado e corrigido.

📋 Resumo das Correções Aplicadas

1. CustomFieldsManager.tsx

Problema: Linhas setHasUnsavedChanges(true) comentadas mas não removidas

Correção: Removidas as linhas que marcavam campos como "não salvos"

// ✅ CORREÇÃO: NÃO marcar hasUnsavedChanges para operações de campo
// Campos customizados são salvos via API própria, não devem afetar estado do pipeline
// setHasUnsavedChanges(true); // REMOVIDO: Causa fechamento automático do modal

2. ModernPipelineCreatorRefactored.tsx

Problema: handleFieldsUpdate chamava markFormDirty() sempre que campos eram atualizados

Correção: Removida a chamada que marcava o formulário principal como "dirty"

const handleFieldsUpdate = useCallback((custom_fields: LocalCustomField[]) => {
  setFormData(prev => ({ ...prev, custom_fields }));
  // ✅ CORREÇÃO CRÍTICA: NÃO marcar formulário como dirty para campos customizados
  // if (pipeline?.id) markFormDirty(); // REMOVIDO: Causa fechamento automático do modal
}, [pipeline?.id]);

🔍 Análise da Causa Raiz

O problema ocorria no seguinte fluxo:

  1. Usuário clica para adicionar/editar campo customizado
  2. Campo é salvo via API (POST/PUT)
  3. handleSaveField chama onFieldsUpdate(updatedFields)
  4. onFieldsUpdate (que é handleFieldsUpdate) chama markFormDirty()
  5. markFormDirty() define hasUnsavedChanges=true no formulário principal
  6. ModernPipelineCreatorRefactored detecta mudanças não salvas
  7. Modal é fechado automaticamente com "Pipeline atualizada com sucesso"

✅ Comportamento Esperado Após Correção

  • Modal permanece aberto ao adicionar novo campo customizado
  • Modal permanece aberto ao editar campo existente
  • Modal permanece aberto ao excluir campo customizado
  • Campos customizados são salvos individualmente via sua própria API
  • Formulário principal não é marcado como "dirty" por operações de campos
  • Apenas mudanças reais no pipeline (nome, descrição, etapas) marcam como "não salvo"

🧪 Como Testar

  1. Acesse o sistema e faça login
  2. Vá para uma pipeline existente e clique em "Editar"
  3. Navegue para a aba "Campos"
  4. Clique em "Adicionar Campo Customizado"
  5. Verifique: Modal permanece aberto após salvar
  6. Clique no ícone de editar de um campo existente
  7. Verifique: Modal permanece aberto após salvar
  8. Clique no ícone de excluir de um campo
  9. Verifique: Modal permanece aberto após excluir
